# 所感

現状、Props, Components, Themesの責務がバラバラなように感じている。これらをきれいに分離するために、このタイミングで一旦各層の役割を整理していく。

# アーキテクチャの再解釈

## Props

- Sass内の全体で使用するprefixの定義
  - prefixはある程度上書きを可能とする。
- Breakpointの定義
  - 各Breakpointの名前やタイミングは初期値だけ定義し、後からSassで上書き可能にする。
- Colorの定義
  - Semanticsの定義
  - SemanticLightnessの定義
  - SemanticLevelsの定義
  - HUEの定義
  - HUELightnessの定義
  - HUELevelsの定義
  - Neutralの定義
  - Mixinの定義と呼び出し

※ 名前やプロパティは初期値だけを定義し、後からSassで上書き可能にする。

## Components

- 各コンポーネントのCSS変数定義
- 各コンポーネントのCSS変数をCSS要素にマッピング

## Themes

- CSS変数（Sass変数）の上書き
- その他、各テーマ専用のコンポーネント実装など

# Color

## クラスなどの生成

以下の要素が存在するので、それらを元に動的に生成する。

- 名前
- 色相
- 彩度
- 輝度リスト
  - 輝度名
  - 輝度値
- レベルリスト
  - レベル名
  - 輝度名
- アクセント切り替わりタイミング輝度値

これらのうち、名前、輝度名、レベルリストや輝度リストの要素数はCSS側から変更できないが、その他の値に関してはCSS変数にしておいてCSSから変更可能とする。

# ファイルの書き方

以下のセクションに分ける。

- Overridable Variants: 上書き可能な変数
- Constant Variants: 上書き不可能な変数
- Mixins: Mixin
- Process: 実際にCSSを出力する場所

こうすることによって、外から呼び出すときに上書き可能な変数を簡単に見分けやすくする。
