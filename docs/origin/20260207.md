# 経緯
以下の点を満たすCSSライブラリが欲しかった。

- Node.jsだけでなく、ASP.NET CoreやPHPなど、様々なフレームワークで利用できるプレーンなCSSライブラリであること
- TailwindやBootstrapなどのようにライブラリへの依存度が高くならないこと
- 古いブラウザでも動作すること
- 複数のテーマを切り替えられるような設計であること
- テーマが確定したら大きくカスタマイズできないこと

# 試したこと
## Bootstrapのカスタマイズ
Sassを上書きしてカスタマイズを行ってみたが、Sassをコンパイルしたタイミングでカラーが固定されてしまう個所があり、カスタマイズ性が低くなってしまった。あと、Bootstrap感の強いデザインを抜け出すことができなかった。

## OpenProps / OpenProps UI
OpenProps、OpenProps UIを使用してカスタマイズを行おうとした。開発体験は非常に良かったものの、OKLCHなど新しすぎる技術が頻出しており、古いブラウザでの動作が怪しかった。また、OpenProps UI自体がまだ発展途上であり、コンポーネントの数も少なかった。

# やろうとしていること
上記の問題を解決するために、独自のCSSライブラリを作成することにした。以下のような構想をもって開発を行う。

## 階層化
以下の階層に分けて実装を行う。
- トークン層: カラーやスペーシングなどのデザインシステムのトークンを定義する
- コアコンポーネント層: ボタンやカードなどの基本的なコンポーネントを定義する
- テーマ層: 複数のテーマを定義し、トークンやコンポーネントのスタイルを上書きする

以下のような名前でパッケージを分ける予定。
- @css-std/props
- @css-std/core
- @css-std/theme-<theme-name>
